## §5. 予測モデル

### §5.1 符号規約

- **収支 d = 消費kcal − 摂取kcal**。d > 0（黒字）はカロリーを削減できた日＝減量に有利、d < 0（赤字）はオーバー。
- **累積収支 cum** = その期間の d の累積和。
- **予実乖離** = 実測体重 − 予測体重。正なら「予測より重い＝ペースが遅れている」、負なら「予測より軽い＝先行」。
- 実装: `lib/data.ts`（`DayData.d` / `DayData.cum`）、`components/ForecastTab.tsx`（`TrajectoryChart` の予実乖離表示）。

### §5.2 個人係数 k の算出方法

個人係数 k（kcal/kg）は「体重を 1kg 動かすのに必要なカロリー収支」を表す。理論値（脂肪組織の熱量）は約 7,700 kcal/kg だが、実際の体組成変化（水分・筋肉量を含む）は個人差が大きいため、本アプリは既定値 **k₀ = 7,200** から出発し、実測データが十分蓄積した段階で個人ごとに回帰キャリブレーションする。

#### §5.2.1 現行実装（v1.0）— 最小二乗回帰 + 閾値ゲート

**実装箇所**: `app/page.tsx`（回帰計算・キャリブレーション判定）、`components/ForecastTab.tsx`（散布図表示・期間参考値）。

**入力データ**: 体重が実測されている日（`w > 0`）の `(cum, w)` ペア（累積収支・体重）。

**回帰**: 単純最小二乗（OLS）で `w = slope × cum + intercept` を推定し、`k_raw = −1 / slope` とする。

**キャリブレーション条件**（すべて満たす場合のみ実測値を採用、それ以外は既定値 7,200 を使用）:
| 条件 | しきい値 |
|---|---|
| 測定日数 | ≥ 10日 |
| 測定期間の幅 | ≥ 21日（暦日、連続でなくてよい） |
| 算出値の妥当域 | 4,000 ≤ k_raw ≤ 12,000 kcal/kg |

採用時は `k = round(k_raw / 50) × 50`（50kcal刻みに丸め）。

**既知の弱点**:
- OLS は外れ値（測定誤差・水分変動による体重の急な跳ね）に弱く、少数の外れ値で傾き（＝k）が大きく歪みうる。
- 体重の生値をそのまま回帰に使っており、日々のノイズ（水分・食事・測定タイミングで ±1〜2kg）を平滑化していない。
- 表示期間を絞った際（`ForecastTab` の「体重×累積黒字」カード、10日/30日窓）は測定点数が減り、上記の弱点がより顕著に出る。この期間別kは**参考値表示のみ**で、予測（トラジェクトリ・ホーム）には全期間の較正済みkを使用する（`ForecastTab.tsx` の `windowK` / `kVal` の分離を参照）。

#### §5.2.2 改訂方針（v2.0）— ロバスト統計手法への置き換え

v1.0 の弱点（外れ値脆弱性・ノイズ未処理）に対応するため、体重時系列の解析で確立された手法に置き換える。個々の手法は独立して採用実績があり、組み合わせて使うことで日次ノイズと外れ値の双方に頑健なトレンド・傾き推定を行う。

**設計方針の出典・根拠**:
1. **The Hacker's Diet**（John Walker）が提唱し、TrendWeight・Libra・Happy Scale 等の体重トラッキングアプリが採用する **EWMA（指数加重移動平均）** による体重トレンド抽出。日々の水分変動を均し、脂肪トレンドのみを残す。
   出典: [The Hacker's Diet — Signal and Noise](https://www.fourmilab.ch/hackdiet/e4/signalnoise.html), [TrendWeight Help/FAQ](https://trendweight.com/help/)
2. **Theil–Sen 推定量**（全点ペアの傾きの中央値）による傾き推定。破綻点（breakdown point）約29.3%で、外れ値に対して最小二乗より大幅に頑健。線形トレンド推定で最も広く使われるノンパラメトリック手法とされる。
   出典: [Theil–Sen estimator (Wikipedia)](https://en.wikipedia.org/wiki/Theil%E2%80%93Sen_estimator), [The Theil-Sen robust estimator (SAS Blog)](https://blogs.sas.com/content/iml/2019/05/28/theil-sen-robust-regression.html)
3. **Hampel フィルタ / MAD（中央絶対偏差）** による外れ値検出。移動窓内の中央値からの乖離を MAD でスケールし、しきい値（一般に 3×MAD 相当）を超える点を外れ値として棄却・置換する。平均・標準偏差ベースの手法と異なり、外れ値自体による汚染を受けにくい。
   出典: [Outlier Detection with Hampel Filter (Towards Data Science)](https://towardsdatascience.com/outlier-detection-with-hampel-filter-85ddf523c73d/), [A Novel Approach to Speed Up Hampel Filter for Outlier Detection (MDPI Sensors, 2025)](https://www.mdpi.com/1424-8220/25/11/3319)

**処理パイプライン**（体重時系列 → k）:

```
生の体重時系列
  → ① 生理的妥当性クリップ（前日比の異常な変化を除外・クリップ）
  → ② Hampel/MAD による外れ値検出・置換（移動窓の中央値 ± n×MAD）
  → ③ EWMA 平滑化（α ≈ 0.1、日々の水分変動を均す）
  → ④ Theil-Sen ロバスト回帰（累積収支 vs 平滑化体重の傾き）
  → ⑤ 妥当域クランプ + 前回値フォールバック
  → k
```

**① 生理的妥当性クリップ**: 1日あたりの体重変化が生理的に妥当な範囲を超える場合（水分/測定誤差の可能性が高い）、その日の変化量を上限にクリップする。

**② 外れ値検出（Hampel/MAD）**: 移動窓（例: 前後数日）内で中央値と MAD を算出し、`|値 − 中央値| > n × MAD × 1.4826` の点を外れ値として中央値で置換する。

**③ EWMA平滑化**: `trend[t] = trend[t-1] + α × (raw[t] − trend[t-1])`。α が小さいほど平滑（ノイズ除去↑・追従性↓）。平滑化後の系列を「トレンド体重」として以降の回帰に使う。

**④ Theil-Sen 回帰**: トレンド体重と累積収支の全点ペアの傾きの中央値を採用し、`k_raw = −1 / median(slope)` を算出。

**⑤ 妥当域クランプとフォールバック**:
- 算出値が妥当域（v1.0 と同じ 4,000〜12,000 kcal/kg）を外れた場合、または収支と体重変化の符号が矛盾する（＝モデル破綻の兆候）場合は、**前回採用値（`kPrev`）を維持**する（既定値へ直接落とすのではなく、直前の安定値を保つ）。
- 初回（`kPrev` 未設定）は既定値 7,200 にフォールバック。
- 体重がほぼ動いていない場合（変化が閾値未満）も `kPrev` を維持し、ノイズによる誤ったk更新を防ぐ。

**可変窓（ハイブリッドウィンドウ）**: 直近の測定密度に応じて回帰に使う窓の長さを調整する（データが疎な期間は窓を広げて点数を確保し、密な期間は直近に絞ってトレンドの鮮度を保つ）。窓の上限は56日程度を想定。

**診断シグナル**: 算出結果に「クランプが発生したか」「符号矛盾が検出されたか」を示すフラグと理由を付随させる。これは UI 上のエラー表示には使わず、ログおよび AI アドバイザー（§4.2 参照）への文脈情報として扱う——「モデル破綻の兆候」を検知した旨をアドバイザーの分析材料に含めることで、より的確な原因説明（記録漏れの疑い等）に繋げる。

**移行時の非互換点**（v1.0 → v2.0 実装時に判断が必要な事項）:
- v1.0 は k をクライアント（`app/page.tsx`）で全データ再計算しているが、v2.0 は EWMA の逐次更新・前回値保持（`kPrev`）を要するため、**サーバ側で永続化した状態**（前回採用k・更新時刻・診断フラグ）を持つ設計に変える必要がある。
- 永続化が必要な項目: 前回採用k値、最終更新日時、算出に使った窓の日数（診断用）、クランプ/符号矛盾の発生有無とその理由（診断用）。
- 上記はユーザー単位で保持する（マルチユーザー対応後は `user_id` スコープ必須）。
